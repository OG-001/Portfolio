Architecture

System Diagrams

From a 5-layer microfrontend system to an enterprise Next.js component tree.

Ragbrain — Full System
MultiChat — System
MultiChat — Frontend Tree

5-layer architecture: Cloudflare CDN → Single-SPA Shell → 5 MFEs (Module Federation) → Axios API Client → Backend Services (LLM / RAG / PostgreSQL).

🔷

Single-SPA Orchestration

Shell registers each MFE via registerApplication(). MFEs mount/unmount independently without page reloads.

⚙

Module Federation

Each MFE exposes ./App. Shared singletons (react, react-dom) declared eager to prevent version conflicts.

🔐

Auth Bridge Pattern

Auth0 only in shell. Exposes window.shellAuth — MFEs access tokens without importing Auth0 directly.

☁

Cloudflare Cache Strategy

Bundles: immutable, 1yr. Remote entry files: no-cache, max-age=0. Maximum efficiency + instant MFE updates.

Four-layer architecture: Next.js client → FastAPI gateway → Backend services → Data & AI layer.

React component tree: 🆕 = new files, ✏️ = modified files contributed by the frontend engineer.
