Challenges & Solutions

Real problems from Ragbrain — not tutorial-level issues.

01
Challenge

Cross-MFE Authentication Without Coupling

Each MFE needs Auth0 tokens to call the API, but importing @auth0/auth0-react into every MFE creates version conflicts and breaks React Context's singleton requirement.

Solution

Built the Auth Bridge pattern: only the shell imports Auth0 and exposes a typed window.shellAuth object containing getAccessTokenSilently(), isAuthenticated, and logout(). MFEs access tokens via this contract, fully decoupled from the auth library.

02
Challenge

MFE Cache Invalidation on Cloudflare

remoteEntry.js is the critical file the shell fetches to load the latest MFE bundle. Caching it aggressively means stale MFEs; zero cache means unnecessary network trips for all assets.

Solution

Split the cache strategy in _headers: content-hashed bundles get max-age=31536000, immutable (1yr). remoteEntry.js and index.html get no-cache, max-age=0. Zero stale MFEs + maximum cache efficiency simultaneously.

03
Challenge

Streaming LLM Responses in React

Real-time SSE involves managing idle → loading → streaming → error transitions while parsing agent-formatted events (thought / tool / result) with React's async state batching.

Solution

Implemented a useReducer state machine with explicit phases. The reducer accumulates streamed tokens immutably and parses agent event types separately. useRef holds the abort controller for clean cancellation without re-renders.

04
Challenge

LibreChat in a React 18 / Webpack Shell

LibreChat uses React 17 internals and a Vite build system — incompatible with the Webpack-based React 18 shell and Module Federation's shared singleton requirement.

Solution

Created a dedicated librechat-wrapper MFE with its own package.json declaring React 17 and single-spa-react 4. It wraps LibreChat's client and exposes only the container to Single-SPA — version boundary fully isolated within that MFE.
